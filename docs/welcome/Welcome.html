Welcome to CinderISO

This document is intended only to be a brief overview of the CinderISO library. A more comprehensive tutorial will be available soon. There are two examples provided with the library, Simple and Advanced. The code is straight forward and it doesn't take much to get CinderISO going.



Basics

CinderISO meshes (or polygonizes) a group of particles into a polygonal mesh - also known as metaball. The resulting mesh is stored in a Cinder TriMesh. As far as the meshing processing is concerned, there are a few basic properties that drive the overall meshing process. Each of these properties are discussed in the sections to follow.


Grid Size

The grid size controls how many cells are created during the meshing process. Higher grid size values means more cells which means more resolution as wells as more time needed to process.


The maximum grid size that's recommended is 200. Values greater than 200 may result in a memory allocation error during the meshing process.


Particle Radii and Color

CinderISO stores two attributes for each particle, radius and color.


The safe range for radius is between 1.0 and 2.5. Values outside that range will either not show up due to a coarse grid size or will produce undesirable results.


Field Functions

CinderISO only supports a Blinn style field function. More field functions will be added in upcoming releases. The value of a field function is accumulated based on influencing particles. CinderISO uses a maximum search distance to determine how many particles are influencing a particular point. Anything beyond the maximum search distance is negligible.


The maximum search distance defaults to 10.0


Multi-Threading

CinderISO can use one or more threads during the meshing process. Using multiple threads will speed up the meshing process, however the speed up is not linear.
